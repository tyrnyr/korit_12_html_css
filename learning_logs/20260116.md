## 태그 속성 사용하기
- 태그에 추가 정보를 부여할 때 속성을 사용. 태그 옆에 _특정_ 단어를 붙이고 _값_ 을 지정하면 다양한 표현이 가능.

### 태그의 속성(=특성)
- html 태그의 속성을 영어 원문 상 python과 동일하게 attribute를 사용합니다. 그런데 mdn 문서에서 전에 말했던 것처럼 특성이라고 번역하는 중인데, 현재 국내의 대부분의 강의에서 속성이라고 표기하고 있고 현직에서도 그러는 중이라서 속성이라는 표현을 중심으로 강의합니다.
- 예시
    1. `<html lang="ko">`
    2. `<img src="" alt="">`
이상의 예시에서 img 태그는 그림을 넣어주는 태그인데, 원본 파일명이 src 속성에 들어가고, 만약 그림 파일이 없을 경우 대신 표시할 alt(alternative) 속성이 존재합니다. 근데 이 img 태그 내의 src, alt 속성은 p 태그나 h1 태그에는 존재하지 않습니다. 즉, _태그의 동작을 설정 및 제어하는 것_ 이 **태그의 속성(특성)** 이며, 각 태그마다 설정 가능한 속성들이 있습니다.

# 태그로 여러 요소 나열하기
- 어떤 정보를 나열할 때 목록으로 표기하면 시인성이 높아집니다. 이런 목록을 표기할 때 순서가 없는 태그와 순서가 존재하는 태그로 나뉩니다.
## 목록을 표현하는 태그
- `<ul>`태그는 unordered list의 축약어로 순서가 상관없는 리스트를 만들 때 사용합니다. 예시 먼저 확인하겠습니다.
```html
<ul>
    <li>사과
    <li>딸기
    <li>포도
</ul>
```
<ul>
    <li>사과
    <li>딸기
    <li>포도
</ul>

- `<ol>` 태그의 경우는 ordered list의 축약어로 순서가 필요한 리스트를 만들 때 사용합니다.
```html
<ol>
    <li>알파
    <li>베타
    <li>감마
</ol>
```
<ol>
    <li>알파
    <li>베타
    <li>감마
</ol>

이상의 두 경우에서 나오는 `<li>` 태그는 list item의 축약어로 `<ul> 및 <ol>`태그의 하위 항목을 표기할 때 사용합니다.
    
특히 `<ol>`의 속성
    1. type = "A"로 지정하면  ABCD...
    2. default 값은 12345 ...
    3. type=을 지정했을 경우 start속성도 지정할 수 있는데 index와 달리 1부터 시작합니다.

## 용어와 정의를 나열하는 태그
어떤 리스트를 1:1로 매칭하는 경우가 있다고 가정하겠습니다. 예를 들어 apple/사과, grape/포도 등입니다. 용어:정의 가 1:1 대응도 가능하고, n:n도 가능할 겁니다. 즉, 여러 용어에 하나의 정의를 붙이거나 하나의 단어에 여러 정의가 있을 수 있습니다.

- `<dl>` 태그 : definition list의 축약어로 정의 목록이라고 번역됩니다.
- `<dt>` 태그 : definition term의 축약어로 정의할 용어를 뜻합니다.
- `<dd>` 태그 : definition description의 축약어로 용어의 설명 혹은 의미 해석으로 볼 수 있습니다.

## id 값으로 원하는 위치로 이동하기
`p#target_${타겟_$}*100`
이상의 Emmet에서 알 수 있는 점 : 
1. `#` : 태그 내에 id 속성을 의미함. `.`은 class 속성을 의미
2. `$` : 동적으로 변하는 값을 임의적으로 나타내는 기호
3. `[속성명]` : 임의의 속성명을 적을 때 쓰는 대괄호
4. `{쓸내용}` : 여는 태그와 닫는 태그 내에 작성하게 될 내용을 나타낼 때 쓰는 중괄호

## 태그로 연락처 연결하기
연락처를 연결하기 위해서는 href 속성 내에서 mailto: / tel: 을 이용합니다.
1. `<a href ="mailto:a@test.com">a@test.com</a>`
2. `<a href ="tel:010-1234-5678">10-1234-5678</a>`

* 참조
전화번호 링크를 컴퓨터에서 클릭하면 관련 앱이나 파일 프로그램을 선택하도록 되어있습니다. -> 연결
프로그램으로 뭐 쓸지 선택하는 창이 나올 확률이 높습니다. 하지만 모바일의 경우 전화 기능이나 혹은 email기능으로 바로 연결되는 편입니다. 아이폰 -> mail앱 / 안드로이드 -> gmail 앱 이거나 default 앱으로 연결

이상을 모두 합쳐서 생각했을때 `<a>`태그는 http:// / https:// 뿐만 아니라 전화번호/이메일로의 연결이 가능하다는 점입니다. 그리고 심지어 텍스트를 클릭해서 넘어가는 것 뿐만 아니라 이미지 등 다양한 응용 방법이 있습니다.

- 웹 페이지를 사용할 때 아이디와 패스워드를 입력한다거나 체크하고 고르는 등 사용자로부터 입력 받아야 하는 정보들이 있습니다. 이 때 사용하는 form 태그를 중심으로 수업합니다.

## `<form>` 태그 사용하기
- 사용자로부터 입력을 받기 위한 양식을 작성하는 태그를 통틀어서 지칭하는 용어
### `<form>` 태그 속성

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body bgcolor="5f5f5f">
    <h1>01_formtag.html</h1>
    <form action="./formatag.html" method="get">
        <label for="name">이름</label>
        <input id="name" name="my-name" type="text">
        <br><br>
        <label for="age">나이</label>
        <input id="age" name="my-age" type="number">
        <br><br>
        <button type="submit">제출</button>
        <button type="reset">초기화</button>
    </form>    
</body>
</html>
```

- 이상의 예시 코드에서 사용되는 속성들
    1. `<form>` 태그
        - action
        - method
            - 현재 수준에서 경험해보지는 못했지만 우리가 이름과 나이를 입력했을 때 그 정보를 가져가서 보여주는 결과 페이지에 해당합니다. 현재 수준에서는 form 태그 내에 action / method 속성이 있다는 것만 알아두셔도 됩니다.
    
    2. `<input>` 태그
        - 이건 아예 챕터를 따로 빼서 설명할 예정
        - name : HTML 상에서는 특별한 의미를 갖는게 아니라 서버에서 각 항목을 받을 때 무엇에 대한 값인지 구분할 수 있도록 구분하는 용도에 해당합니다. DB와의 연결성과 관련.

    3. `<label>` 태그 : input 태그와 유기적으로 연결됨. input 태그에 부여한 id속성값을 label 태그에서의 for 속성에서 불러오기 때문입니다.

    4. `<button>` 태그
        - type- submit / reset으로 사전에 정의되어있습니다.
        submit의 경우 입력한 결과값을 최종 서버나 특정 페이지 주소로 보내는, 즉 제출하는 용도. reset은 입력한 내용을 모두 초기화 합니다.

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body bgcolor="5f5f5f">
    <h1>01_formtag.html</h1>
    <form action="./formatag.html" method="get">
        <fieldset>
            <legend>개인신상정보</legend>
            <label for="name">이름</label>
            <input id="name" name="my-name" type="text">
            <br><br>
            <label for="age">나이</label>
            <input id="age" name="my-age" type="number">
            <br>
        </fieldset>
        <label for="age">나이</label>
        <input id="age" name="my-age" type="number">
        <br><br>
        <button type="submit">제출</button>
        <button type="reset">초기화</button>
    </form>    
</body>
</html>
```
