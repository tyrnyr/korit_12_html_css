## 갖다 놓는(배치하는) HTML / 꾸며주는 CSS / 동작시키는 JS
- 저희가 Java를 배우면서 작성한 것은 code라는 표현을 썼지만, HTML + CSS로 작업한 것은 문서라고 합니다. 즉, 얘네 둘을 이용해서 만든 것은 프로그램이 아니라 document에 해당합니다.

- 웹 페이지에 들어갈 요소들을 배치하고 디자인하는 정도인 .html 파일과 .css 파일은 브라우저라는 프로그램(저희는 크롬 사용합니다)에서 불러와서 보여주는 문서의 의미입니다.

- 하지만 JS의 경우 서버로부터 전달 받아 브라우저에서 동작하는 파일에 해당합니다. 하지만 Node.js 등 컴퓨터에 설치된 JS 엔진에서 python / java와 같은 언어처럼 복잡한 알고리즘을 정의하고 컴퓨터에 명령을 내릴 수 있습니다.

이상을 이유로 JS는 프로그래밍 언어이기 때문에 code라고 하고, HTML + CSS는 문서라고 합니다.

## 웹 디자이너 vs. 웹 퍼블리셔
웹디자이너가 작업한 결과물을 HTML + CSS 문서로 작성하는 사람을 우리나라에서는 웹 퍼블리셔라는 표현을 씁니다. 여기에 JS까지 적용하여 기능을 만들어 낼 수 있는 사람을 웹 개발자라고 할 수 있겠습니다.

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>document</title>
</head>
<body>

</body>
</html>
```
`! + tab` 으로 자동 생성된 것은 이상과 같습니다. 이는 웹 개발 시에 기계 또는 시스템에 전달하는 각종 정보로, _HTML 문서 작성 시에 꼭 필요한 내용_ 들을 담고 있습니다. 문서 제목, 작성자, 사용 언어 및 각종 메타 정보(메타 데이터) 등으로 구성되어 있는데, 어차피 형식 자체는 동일하기 때문에 일일이 타이핑할 필요 없이 단축키를 지원합니다. 이를 Emmet이라고 하는데 `! + tab`도 이 중 하나에 속합니다. 앞으로 에밋이라고 했을 때, 특정 문서 작성을 빠르게 해주는 단축키로 생각하시면 되겠습니다. 그리고 `!+tab`은 저희가 .html 파일 생성할 때마다 맨 처음에 하는 작업이 될겁니다.

- `<html lang="en">`의 의미
lang 속성은 HTML 문서의 웹사이트가 어떤 언어로 되어있는지 _컴퓨터에게 알려주는_ 역할을 합니다. 이걸 지정해줬을 때 구글 / 네이버 등에서 검색할 때 한글로 된 웹 사이트를 찾는 것이 가능합니다. 또한 시각 장애인들을 위한 컴퓨터가 웹 사이트를 분석할 때도 쓰입니다. 그리고 맥 사용자의 경우 lang="en"인데 캡스락 눌러서 한글로 웹 페이지 작성하면 기본 서체가 궁서체입니다.

## 첫 태그 사용하기
`<어쩌고>` + `</어쩌고>` : `/`가 있는 쪽은 닫는 태그에 해당함.

```html
<div>
  <p>
    안녕하세요.
  </p>
</div>
```
라고 가정했을 때 div 태그 내에 p 태그가 종속되어있다고 볼 수 있습니다. 이를 확인할 수 있도록 들여쓰기(indent-writing)를 사용하여 다른 개발자들도 쉽게 알아채도록 합니다.

근데 가끔 닫는 태그가 없는 태그도 존재합니다. `<br>`
MDN 공식 문서(HTML 관련) 상에서는 `<br>`을 `<br/>`로 쓰지말 것을 권유하고 있습니다.

근데 리액트에서는 `<br>`로 쓰면 오류나서 `<br/>`로 써야 합니다.

1. `<hr>`: horizontal의 축약어로 가로줄을 의미함.
<hr>

<h1>안녕하세요</h1>
<div>여기는 분할되는 부분입니다.
  <p>그리고 또 안에 들어가는 부분도 쓸 수 있구요.
  </p>
</div>

- `#nbsp;` 강제 공백 / html 문서 상에서는 스페이스 바 / 엔터키 탭 키는 코드의 가독성을 위해서만 사용될 뿐이지 웹페이지 자체에는 기능이 없기 때문에 강제로 공백을 삽입해야 할 때 사용됩니다.
- 하지만 html에서 뭔가 공백 등 꾸미는 역할에 해당하는 부분이 전부 css 쪽으로 이관되었기 때문에 잘 쓰지도 않습니다.

## 텍스트 관련 태그
웹 페이지에서 텍스트를 지정할 때는 내용에 따라 글자를 강조하거나 중요하다고 표시하게 됩니다. 나무위키 같은데 보면 괜한 뻘글 있는 쪽에 취소선 그려져있고 그렇습니다. 이렇게 특정 텍스트의 스타일을 다양하게 조절하는 태그들도 존재합니다.

### 스타일 강조 태그
제목이 아니라 본문 내에서의 세부 스타일 관련
1. `<b>` 태그 : bold를 의미합니다.
2. `<strong>` : '굵게'를 뜻합니다.
3. `<i>` : 이탤릭체를 뜻합니다.
4. `<em>` : '기울임체'를 뜻합니다.

ch01_basic_tags에 strongtag.html 파일을 만들고, lang="ko"로 바꾸시고 body의 bgcolor를 5f5f5f로 만든 다음에 라이브 프리뷰를 켜서 세팅해두시오.

- 옛날에는 HTML과 CSS가 분리되지 않았기 때문에 웹 페이지 상에서의 서체를 굵게 지정하거나 기울이는 것을 전부 다 태그로 처리했었습니다.

- 5 버전 이후부터는 태그는 코드적으로 어떤 위상에 속하는지만 표시하는 지표로 쓰이고, 데코레이션은 CSS로 넘어가게 됩니다.

- 즉, b 태그와 i 태그는 시각적으로 강조하는 효과에 해당하고, strong 태그와 em 태그의 경우에는 해당 text들의 중요 토픽에 해당하는 부분이라는 것을 컴퓨터에게 알려주는 역할을 합니다.

- 그러면 검색 엔진에서 특정 검색어를 지정한다고 할 때, 엔진은 strong이나 em 태그에 검색어와 동일한 부분이 있는지를 체크하게 됩니다.

### 첨자 태그
위 첨자 및 아래 첨자라는 개념을 학습한 적이 있습니다. 보통은 영어 시간이나 수학 시간에 배웁니다.
- 위 첨자 예시
1<sup>st</sup>, 2<sup>nd</sup>, 3<sup>rd</sup>, 4<sup>th</sup>

- 아래 첨자 예시
H<sub>2</sub>O

### 밑줄 태그와 취소선 태그
_이탤릭체_ 쓰거나 **이렇게** 강조하거나 하는 방식이 markdown에서 씁니다.

밑줄 예시
<u>이렇게 쓰면 됩니다</u>.

취소선 예시
<s>이렇게 쓰면 됩니다</s>
